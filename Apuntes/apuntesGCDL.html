☁️ 1. Fundamentos de Cloud y Economía

Modelos de Servicio (Servicio y Control)

  • **IaaS:** Infraestructura (VMs, Red). Máximo control. Ej: Compute Engine.
  • **PaaS:** Plataforma (SO, Runtime, DB). Enfocado en el código. Ej: App Engine.
  • **SaaS:** Aplicación completa. Mínimo control. Ej: Gmail, Workspace.
  • **FaaS:** Funciones (código efímero) disparadas por eventos. Ej: Cloud Functions.

Modelos de Despliegue (Ubicación)

  • **Pública:** Servicios de GCP, escalabilidad global.
  • **Privada:** Datacenter propio, control exclusivo.
  • **Híbrida:** Integra pública y privada (p. ej., mediante Cloud Interconnect).
  • **Multi-cloud:** Uso estratégico de múltiples proveedores de nube.

Valor y Costes

  • **OpEx (Operativo):** Pago por uso, escalabilidad flexible. (Cloud).
  • **CapEx (Capital):** Inversión inicial en hardware. (On-Premises).
  • **FinOps:** Cultura y práctica para gestionar costes de Cloud.
  • **Control:** Uso de **Etiquetas** (Labels) para asignar costes y **Presupuestos** para alertas.

💻 2. Compute: VMs, Contenedores y Serverless

Compute Engine (VM)

IaaS - Máximo control de SO.

  • MIGs (Autoescalado / Autorreparación).
  • SUDs y CUDs para ahorro.

GKE (Kubernetes)

Orquestación de Contenedores.

  • **Autopilot:** Serverless para el plano de control (gestión automática).
  • Ideal para Microservicios.

Cloud Run

Serverless de Contenedores.

  • Escala a cero. Pago por uso exacto.
  • Ideal para APIs y backends web sin complejidad de K8s.

App Engine & Functions

PaaS y FaaS (eventos).

  • **App Engine:** Plataforma web gestionada.
  • **Functions:** Código event-driven, respuesta rápida.

Diseño Resiliente y Global

La clave para la **Alta Disponibilidad (HA)** es la **redundancia**. Despliega en múltiples **Zonas** (dentro de una región) para protegerte contra fallos de hardware o software. Usa múltiples **Regiones** (Multi-regional) para la **Recuperación ante Desastres (DR)** ante fallos regionales.

🗃️ 3. Almacenamiento y Bases de Datos

Almacenamiento (Discos, Archivos, Objetos)

  • **Cloud Storage (Objetos):** Duradero, escalable, ideal para data lakes, backups, web estática.
    • Clases por frecuencia de acceso: Standard (frecuente), Nearline, Coldline, Archive (menos frecuente).
  • **Persistent Disk (Bloques):** Discos de alta velocidad para VMs de Compute Engine (se adjuntan).
  • **Filestore (Archivos):** Servicio gestionado de NFS para cargas de trabajo empresariales que requieren compartición de archivos.

Bases de Datos (SQL vs NoSQL)

Relacional (ACID):

  • **Cloud SQL:** Managed MySQL, PostgreSQL, SQL Server. Fácil de usar, escalamiento vertical.
  • **Cloud Spanner:** Base de datos relacional global, escalable horizontalmente y con **consistencia fuerte**. Para cargas críticas de escala masiva.

NoSQL (Escalable, Flexible):

  • **Cloud Bigtable:** Wide-column de alto rendimiento. Ideal para IoT y series de tiempo.
  • **Firestore:** Documentos. Sincronización en tiempo real para aplicaciones móviles y web.

📈 4. Analítica, Streaming y Machine Learning

Data Engineering y Analítica

  • **BigQuery:** Data warehouse **serverless** para petabytes. Permite analítica a escala mediante SQL.
  • **Pub/Sub:** Capa de mensajería asíncrona (eventos) para **desacoplar** sistemas. Clave en arquitecturas de streaming.
  • **Dataflow:** Procesamiento de datos **streaming y batch** de forma unificada. Altamente escalable y gestionado.
  • **Dataproc:** Clústeres Hadoop/Spark gestionados. Ideal para migrar ecosistemas existentes.

Inteligencia Artificial y ML

  • **Vertex AI:** Plataforma unificada para todo el ciclo de vida de ML (desde el experimento hasta el despliegue).
  • **BigQuery ML (BQML):** Construir modelos de ML (p. ej., forecasting, clustering) directamente usando sintaxis SQL.
  • **AutoML:** Permite a usuarios con poca experiencia en ML crear modelos de alta calidad para casos de uso comunes (Visión, NLP).
  • **APIs Preentrenadas:** (Vision AI, Natural Language, Translation). Ofrecen capacidades de IA listas para usar.

🛡️ 5. Seguridad, IAM y Red

Identidad y Gobernanza

  • **Cloud IAM:** Define el acceso: **Miembro, Rol, Recurso**.
  • **Mínimo Privilegio:** Principio fundamental de seguridad (dar solo lo necesario).
  • **Políticas de Organización:** Gobernanza centralizada (restricción de regiones, desactivación de servicios).

Protección y Cifrado

  • **Cifrado por Defecto:** Todos los datos en reposo son cifrados (Google gestiona las claves).
  • **Cloud KMS:** Servicio gestionado de claves. Permite usar **CMEK** (claves gestionadas por el cliente).
  • **Security Command Center:** Vista unificada de riesgos, vulnerabilidades y cumplimiento.
  • **Cloud Armor:** WAF y mitigación de ataques **DDoS** en el borde de la red.

Red y Conectividad

  • **VPC:** Red global. Las subredes son regionales.
  • **Cloud Load Balancing:** Balanceador de carga global, mejora el rendimiento y HA.
  • **Cloud Interconnect:** Conexión dedicada de alta velocidad On-premises a GCP.
  • **Anthos:** Plataforma de gestión para entornos **híbridos** y **multi-cloud**.

⚙️ 6. Observabilidad, SRE y Optimización de Costes

SRE y Observabilidad

  • **Observabilidad:** Logging, Monitoring, Trace, Profiler, Debugger (Visión 360° del sistema).
  • **SLO (Service Level Objective):** El objetivo de disponibilidad que promete el equipo al usuario (lo que se mide).
  • **Presupuesto de Error:** La tolerancia al fallo permitida (tiempo que se puede incumplir el SLO).
  • **Automatización del Toil:** Eliminar el trabajo manual para centrarse en la ingeniería.

Optimización de Costes (FinOps)

  • **CUDs (Committed Use Discounts):** Descuentos por comprometerse a un uso constante (1 o 3 años). Máximo ahorro.
  • **SUDs (Sustained Use Discounts):** Descuentos automáticos por usar VMs durante gran parte del mes.
  • **VMs Preemptibles (Spot VMs):** VMs baratas para cargas tolerantes a interrupciones (p. ej., batch).
  • **Recomendador:** Ofrece sugerencias para redimensionar VMs subutilizadas y aplicar descuentos.

🧠 7. Preguntas y Respuestas del Examen (Resumen)

Fundamentos y Economía

  • Propuesta de valor: reducir costes y escalar bajo demanda con pago por uso.
  • CapEx a OpEx: convertir compras de hardware en tarifas de servicio.
  • TCO: suele disminuir al optimizar consumo y gobernanza en nube.
  • Transformación digital: integrar tecnología para cambiar procesos y cultura.
  • Elasticidad: escalado automático de recursos según la demanda.
  • Latencia vs ancho de banda: latencia es demora; ancho de banda es capacidad.

Modelos de Servicio

  • IaaS: proveedor entrega infraestructura; cliente gestiona SO y apps.
  • PaaS: proveedor gestiona plataforma y herramientas; cliente apps y datos.
  • SaaS: proveedor gestiona toda la aplicación; el cliente solo usa y configura.
  • Responsabilidad compartida: al pasar a SaaS, aumenta responsabilidad del proveedor.

Red y Conectividad

  • DNS: traduce nombres de dominio a direcciones IP para el ruteo.
  • Fibra óptica: más datos y velocidad a largas distancias con menor pérdida.

Cumplimiento y Residencia

  • Residencia de datos: elegir proveedor público que garantice ubicación regional.
  • Entrega global: servir contenido mundialmente manteniendo almacenamiento en la región requerida.

Compute y Licenciamiento

  • Picos temporales (2 semanas): usar VM bajo demanda, no compromisos a 1–3 años.
  • Windows Server: optimizar con licenciamiento PAYG en Compute Engine.

Seguridad y Gobernanza

  • Identificar VMs vulnerables: usar Security Command Center.
  • Proveedor nube pública: mantenimiento de hardware y gestión de capacidad.

Arquitectura y Latencia

  • Redundancia con baja latencia: distribuir VMs en distintas zonas de una misma región.
  • Característica clave de cloud: entrega bajo demanda vía Internet con autoservicio.

🏛️ 8. Jerarquía de Recursos y Gobernanza

Estructura

  • Organización → Carpetas → Proyectos → Recursos.
  • Herencia: IAM y Políticas se propagan hacia abajo.
  • Separación: proyectos por entorno (dev/test/prod) y dominio.

Políticas de Organización

  • Restringir regiones, tipos de recursos y servicios permitidos.
  • Aplicar controles globales de cumplimiento y seguridad.
  • Etiquetas y Tags para gobernanza y coste.

Buenas Prácticas

  • Diseño por dominio: equipos y líneas de producto.
  • Separación de duty: acceso mínimo y revisiones periódicas.
  • Automatizar aprovisionamiento con IaC.

💳 9. Facturación y FinOps Avanzado

Cuenta de Facturación

  • Vincular proyectos a cuentas de facturación.
  • Presupuestos y alertas para evitar sobrecostes.
  • Exportar costes a BigQuery para análisis.

Optimización

  • CUDs (1–3 años), SUDs y Spot VMs.
  • Recommender: rightsizing y apagado de recursos ociosos.
  • Autoescalado y escalado a cero en serverless.

Imputación de Costes

  • Etiquetas y Tags para centros de coste y proyectos.
  • Carpetas por unidad de negocio para reportes.
  • Políticas para prevenir recursos no aprobados.
